API Reference

El backend flexible proporciona endpoints REST para operaciones de base de datos con autenticación JWT.

Autenticación y Headers

Todas las rutas de la API requieren autenticación mediante token JWT en el header Authorization. El token se obtiene al iniciar sesión en el endpoint POST /api/auth/login.

Headers Requeridos

Authorization Bearer
Content-Type application/json

Códigos de Respuesta

200 - OK
201 - Creado
400 - Error de validación
401 - No autenticado
403 - Sin permisos
404 - No encontrado
500 - Error interno

Obtener Token JWT

Inicia sesión para obtener el token:

curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@ejemplo.com", "password": "tu_contraseña"}'

Base URL

http://localhost:5000/api

Para uso local con el backend levantado en el puerto 5000.

/api/data

Obtener datos con filtros, paginación y ordenamiento

Query Parameters

Parámetro Tipo Requerido Descripción
table string Sí Nombre de la tabla (ej: productos, solicitudes)
filters JSON No Filtros en formato JSON con operadores
limit number No Máximo de registros a retornar

POST
/api/data/query

Consultas SQL personalizadas (solo lectura, con validación de seguridad)

Body (JSON)

Campo Tipo Descripción
query string Consulta SQL (solo SELECT permitido)
params array Parámetros para la consulta parametrizada

Solo se permiten consultas SELECT. Comandos como DROP, ALTER, DELETE, INSERT, UPDATE están bloqueados por seguridad.

Ejemplo de Body (JSON de Entrada)

{
  "query": "SELECT p.id, p.destino, p.precio, a.nombre AS agencia FROM productos p JOIN agencies a ON a.id = p.agencia_id WHERE p.precio > $1 AND p.disponibilidad > $2 ORDER BY p.precio ASC",
  "params": [500, 0]
}

Ejemplo de Petición (curl)

curl -X POST "http://localhost:5000/api/data/query" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -H "Content-Type: application/json" \
  -d '{
    "query": "SELECT * FROM productos WHERE precio > $1 ORDER BY precio ASC LIMIT 5",

Configuración en Postman

  1. 1 Abre Postman y crea una nueva Collection llamada "Sistema de Cupos API".
  2. 2 Ve a la pestaña Variables de la colección y configura:
Variable Initial Value Current Value
base_url http://localhost:5000 http://localhost:5000
token (dejar vacío) (se obtiene al hacer login)
  1. 3 En la pestaña Authorization de la colección, selecciona Bearer Token y en el campo Token ingresa {{token}} (referencia a la variable).
  2. 4 Crea una request Login dentro de la colección:

Method: POST  |  URL: {{base_url}}/api/auth/login

Body (raw JSON): {"email": "admin@ejemplo.com", "password": "tu_contraseña"}

  1. 5 En la pestaña Tests de la request Login, pega este script para guardar automáticamente el token: 
// Guardar token JWT automáticamente después del login
if (pm.response.code === 200) {
    var jsonData = pm.response.json();
    pm.collectionVariables.set("token", jsonData.token);
    console.log("✅ Token guardado exitosamente");
}
  1. 6 Crea el resto de las requests (GET /api/data, POST /api/data, etc.) usando {{base_url}}/api/data como URL. Todas heredarán el token de la colección.
  2. 7 Ejecuta primero Login para obtener el token, luego el resto de las requests funcionarán automáticamente.
