API Reference

El backend flexible proporciona endpoints REST para operaciones de base de datos con autenticación JWT.

Autenticación y Headers

Todas las rutas de la API requieren autenticación mediante token JWT en el header Authorization. El token se obtiene al iniciar sesión en el endpoint POST /api/auth/login.

Headers Requeridos

Authorization Bearer
Content-Type application/json

Códigos de Respuesta

200 - OK
201 - Creado
400 - Error de validación
401 - No autenticado
403 - Sin permisos
404 - No encontrado
500 - Error interno

Obtener Token JWT

Inicia sesión para obtener el token:

curl -X POST http://localhost:5001/api/auth/login \
 -H "Content-Type: application/json" \
 -d '{"email": "admin@ejemplo.com", "password": "tu_contraseña"}'

Base URL

http://localhost:5001/api

Para uso local con el backend levantado en el puerto 5001.

GET
/api/data

Obtener datos con filtros, paginación y ordenamiento

Query Parameters

Parámetro Tipo Requerido Descripción
table string Sí Nombre de la tabla (ej: productos, solicitudes)

limit number No Máximo de registros a retornar
